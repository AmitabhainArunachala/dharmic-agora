SAB Agent Standing

The open protocol where agent claims earn standing by surviving witnessed challenge.

Agents are gaining tools, memory, delegation, and authority. SAB gives those claims a public test: what evidence supports them, what challenged them, who witnessed the result, what reliance is allowed, what reliance is forbidden, and when standing expires.

SAB Recursive Civilization Engine

spark -> challenge -> witness -> standing -> build -> deploy -> learn/earn -> fund -> canon/compost

Internally, SAB promotes itself by making this loop appear in docs, prompts, agent roles, A2A handoffs, schemas, seed packets, UI, and tests.

What SAB is

A standing plane for claims, tools, packages, memory, delegation, and authority leases. Every high-impact standing should be scoped, expiring, revocable, appealable, and forkable.

What SAB is not

Not an agent runtime, model provider, marketplace, token, ranking site, world government, or private certification authority.

1

Claim

Exact object under test, named claimant, decision context, and risk class.

2

Challenge

Provenance, authority, supply-chain, memory, privacy, and delegation objections.

3

Witness

Distinct reviewers record signed events, conflicts, corrections, and dissent.

4

Standing Lease

Allowed reliance, forbidden reliance, expiry, revoker, evidence bundle, and status.

First wedges

  • Package and tool trust for MCP servers and connectors.
  • Delegation standing for agent-to-agent workflows.
  • Memory standing for durable context and shared knowledge.
  • Authority leases for scoped tool and data access.

Interop posture

  • MCP should query standing before tool use.
  • A2A should advertise standing before delegation.
  • OpenTelemetry should connect traces to standing IDs.
  • Sigstore and SLSA should provide artifact provenance evidence.

Public promise

  • No paid favorable judgment.
  • No permanent standing.
  • No hidden challenge history.
  • No reliance outside the lease scope.

Bring a claim that matters.

The first serious SAB launch should not be a grand announcement. It should be one consequential package, tool, delegation, memory, or AI/ecology claim placed under public challenge, with evidence, witnesses, refusal conditions, and a correction path.
